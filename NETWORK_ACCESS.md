# Доступ к приложению с других устройств

## Проблема
При попытке открыть тест с другого устройства (телефон, планшет) через сетевую ссылку приложение может не работать.

## Возможные причины и решения

### 1. Брандмауэр Windows блокирует подключения

**Решение:**
1. Откройте **Панель управления** → **Система и безопасность** → **Брандмауэр Защитника Windows**
2. Нажмите **Разрешить взаимодействие с приложением или компонентом**
3. Нажмите **Изменить параметры**
4. Найдите **Node.js** в списке или нажмите **Разрешить другое приложение**
5. Убедитесь, что отмечены галочки для **Частная** и **Публичная** сети

**Или добавьте правило вручную:**
```
Откройте PowerShell от имени администратора и выполните:
netsh advfirewall firewall add rule name="Node.js Server" dir=in action=allow protocol=TCP localport=5000
```

### 2. Устройства находятся в разных сетях

**Проверка:**
- Убедитесь, что компьютер и мобильное устройство подключены к **одной и той же Wi-Fi сети**
- Проверьте, что не используется гостевая сеть (Guest Network)

### 3. Антивирус блокирует подключения

**Решение:**
- Временно отключите антивирус и проверьте доступ
- Если помогло, добавьте Node.js в исключения антивируса

### 4. Неправильный IP-адрес

**Как узнать правильный IP-адрес:**

1. **Windows:**
   ```
   Win + R → cmd → ipconfig
   Ищите "IPv4-адрес" для вашего Wi-Fi адаптера
   ```

2. **Проверьте в терминале сервера:**
   При запуске сервер выводит все доступные IP-адреса

3. **Используйте ручной ввод IP:**
   В интерфейсе приложения есть поле для ручного ввода IP-адреса

### 5. Сервер запущен не на всех интерфейсах

**Проверка:**
Убедитесь, что сервер запущен с `0.0.0.0`:
```javascript
app.listen(PORT, "0.0.0.0", () => { ... });
```

Это уже исправлено в текущей версии.

### 6. Порт 5000 занят другим приложением

**Решение:**
```
Измените порт в файле .env или при запуске:
PORT=3000 npm start
```

## Проверка работы сервера

### С компьютера:
1. Откройте браузер
2. Перейдите на `http://localhost:5000`
3. Если открывается - сервер работает локально

### С другого устройства:
1. Убедитесь, что оба устройства в одной сети
2. Узнайте IP-адрес компьютера (например: 192.168.0.105)
3. На мобильном устройстве откройте `http://192.168.0.105:5000`

## Тестирование подключения

### 1. Ping-тест
На мобильном устройстве установите приложение для проверки сети и попробуйте пинговать IP компьютера.

### 2. Проверка в браузере
Откройте `http://[IP]:5000/api/test` - должно вернуться `{"message":"Сервер работает! Привет от бэкенда!"}`

### 3. Проверка логов
Запустите сервер и следите за логами - вы увидите все входящие запросы.

## Дополнительные советы

1. **Используйте Chrome на Android** - лучшая совместимость
2. **Отключите VPN** на обоих устройствах
3. **Перезагрузите роутер** если ничего не помогает
4. **Попробуйте создать мобильную точку доступа** на телефоне и подключите к ней компьютер

## Если ничего не помогает

Используйте туннелирование (ngrok, localtunnel):

```bash
npm install -g localtunnel
lt --port 5000
```

Это создаст публичную ссылку, доступную из любой сети.
